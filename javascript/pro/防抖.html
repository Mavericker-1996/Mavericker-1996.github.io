# 防抖原理

防抖的原理就是：你尽管触发事件，但是我一定在事件触发 n 秒后才执行，如果你在一个事件触发的 n 秒内又触发了这个事件，那我就以新的事件的时间为准，n 秒后才执行，总之，就是要等你触发完事件 n 秒内不再触发事件，我才执行，真是任性呐!

首先我们先看一下这里用到的setTimeout函数的返回内容：

返回值timeoutID是一个正整数，表示定时器的编号（默认从1开始）。这个值可以传递给clearTimeout()来取消该定时。传入一个错误的 ID 给 clearTimeout()不会有任何影响；也不会抛出异常。 在同一个对象上（一个window或者worker），setTimeout()或者setInterval()返回的定时器编号不会重复。但是不同的对象使用独立的编号池。

# 初试牛刀

根据上面的描述，我们可以写第一版的代码，思路如下：

  1. 首先确定输入内容和输出内容，输入有两个参数：需要防抖的函数func，需要等待的时间wait。输出应该是一个函数。
  2. 我们该如何实现如果在wait秒内再次触发的时候被覆盖呢？这里就用到的前面讲到的clearTimeout函数和闭包的知识。

结合这两点，代码如下：

// 第一版
function debounce(func, wait) {
    var timeout;
    return function () {
        clearTimeout(timeout)
        timeout = setTimeout(func, wait);
    }
}

# this和event对象

现在我们有两个问题需要解决：this，event对象参数。

在本例中getUserAction函数不使用debounce函数的时候，this是指向<div id='container'></div>dom元素的。且可以接受一个event对象作为参数。当然可以接受更多参数，如果你需要的话。

由于setTimeout讲函数作为参数传递时会发生隐式this绑定丢失，所以我们可以用箭头函数加以修复。

关于参数问题我们可以用apply函数和arguments加以解决。

综上我们就可以写出我们的第二版debounce函数：

function debounce(func, wait) {
  var timeout;
  return function() {
    clearTimeout(timeout);
    timeout = setTimeout(() => {
      func.apply(this, arguments);
    }, wait);
  };

# 节流

# 节流原理

节流的原理很简单：

如果你持续触发事件，每隔一段时间，只执行一次事件。也就是说，我会把你在规定时间内的所有请求合并为一次请求。

个人理解 函数节流就是fps游戏的射速，就算一直按着鼠标射击，也只会在规定射速内射出子弹。

根据首次是否执行以及结束后是否执行，效果有所不同，实现的方式也有所不同。

关于节流的实现，有两种主流的实现方式，一种是使用时间戳，一种是设置定时器。

# 时间戳

让我们来看第一种方法：使用时间戳，当触发事件的时候，我们取出当前的时间戳，然后减去之前的时间戳(最一开始值设为 0 )，如果大于设置的时间周期，就执行函数，然后更新时间戳为当前的时间戳，如果小于，就不执行。

function throttle1(func, wait) {
  var prev = 0;
  return function() {
    var curr = new Date();
    if (curr - prev > wait) {
      func.apply(this, arguments);
      prev = curr;
    }
  };
}

# 定时器

当触发事件的时候，我们设置一个定时器，再触发事件的时候，如果定时器存在，就不执行，直到定时器执行，然后执行函数，清空定时器，这样就可以设置下个定时器。

function throttle2(func, wait) {
  var timeout;
  return function() {
    if (!timeout) {
      timeout = setTimeout(() => {
        timeout = null;
        func.apply(this, arguments);
      }, wait);
    }
  };
}

所以比较两个方法：

  1. 第一种事件会立刻执行，第二种事件会在 n 秒后第一次执行
  2. 第一种事件停止触发后没有办法再执行事件，第二种事件停止触发后依然会再执行一次事件

# 两者综合一下

那我们想要一个什么样的呢？

有人就说了：我想要一个有头有尾的！就是鼠标移入能立刻执行，停止触发的时候还能再执行一次！

当第一次触发的时候，立即执行。接着会将第一个100s内的所有请求合并为一个，之后也是如此。

所以我们综合两者的优势，然后双剑合璧，写一版代码，当一个进行时，就把另一个初始化。

function throttle3(func,delay){
	var start = 0,timeoutID,context,args;
	var later = function(){
		console.log('later');
		start = new Date();
		timeoutID = null;
		func.apply(context,args);
	}
	return function(){
		var end = new Date();
		context = this;
		args = arguments;
		var remain = delay - (end-start);
		if(remain<=0||remain>delay){
			console.log(remain);
			if(timeoutID){
				clearTimeout(timeoutID);
				timeoutID = null;
			}
			start = end;	
			func.apply(context,args);
		}else if(!timeoutID){
			timeoutID = setTimeout(later,delay);
		}
	}
}
