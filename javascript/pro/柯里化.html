# 柯里化

# 定义

维基百科中对柯里化 (Currying) 的定义为：

In mathematics and computer science, currying is the technique of translating the evaluation of a function that takes multiple arguments (or a tuple of arguments) into evaluating a sequence of functions, each with a single argument.

翻译成中文：

在数学和计算机科学中，柯里化是把接受多个参数的函数变换成接受一个单一参数（最初函数的第一个参数）的函数，并且返回接受余下的参数而且返回结果的新函数的技术。

举个例子：

function add(a, b) {
    return a + b;
}

// 执行 add 函数，一次传入两个参数即可
add(1, 2) // 3

// 假设有一个 curry 函数可以做到柯里化
var addCurry = curry(add);
addCurry(1)(2) // 3

# 初次尝试

我们首先可以尝试着写一下乞丐版的curry函数，思路如下：我们可以用一个参数来保存当前传入的参数，然后在返回的函数中再将所有参数还原，然后再进行调用：

function curry(fn){//传入的第一个参数是函数
	var args = [...arguments].slice(1);//先保持当前传入的参数
	return function(){
		var newArgs = [...args,...arguments];//将参数还原
		fn.apply(this,newArgs);//调用原函数
	}
}

# 第二次尝试

先上代码：

const curry2 = fn =>{
	const _c = (restNum,args) => restNum === 0 ? fn(...args) : x =>
    _c(restNum-1,[...args,x]);//带{}需要return
	return _c(fn.length,[]); //递归开始，fn.length返回函数fn的形参
}
const plus = curry2((a,b,c,d,e)=>a+b+c+d+e);
console.log(plus(1)(2)(3)(4)(5));//15

是不是很简单？现在我们来梳理一下思路：

简单思考可以知道，柯里化部分配置函数的步骤数等于 fn 的参数个数，也就是说有两个参数的 plus 函数需要分两步来部分配置。函数的参数个数可以通过fn.length获取。

总的想法就是每传一次参，就把该参数放入一个参数列表 args 中，如果已经没有要传的参数了，那么就调用fn(...args)将原函数执行。要实现这点，我们就需要一个内部的判断函数 _c(restNum, args)，函数接受两个参数，一个是剩余参数个数 restNum，另一个是已获取的参数的列表 args；_c的功能就是判断是否还有未传入的参数，当 restNum 为零时，就是时候通过fn(...args)执行原函数并返回结果了。如果还有参数需要传递的话，也就是说 restNum 不为零时，就需要返回一个单参数函数

x => _c(restNum-1,[...args,x]);//剩余参数-1，同时将传入的参数加入到args中
1

最后我们通过调用_c(fn.length,[])来让递归开始。

这里学到的一个点是箭头函数不带{}是直接return的，如果带有{}的话就需要return一下。

还有就是我们可以通过fn.length来获取函数的形参长度，而arguments的话是用来获取函数传入的实参长度。

当使用console.log，或者进行运算时，隐式转换就可能会发生。从上面三个例子中我们可以得出一些关于函数隐式转换的结论。

当我们没有重新定义toString与valueOf时，函数的隐式转换会调用默认的toString方法，它会将函数的定义内容作为字符串返回。而当我们主动定义了toString/vauleOf方法时，那么隐式转换的返回结果则由我们自己控制了。其中对于+运算符中valueOf的优先级会比toString高一点。

这样一来我们就可以通过返回函数的toString方法，来达到既可以保持返回一个函数，又能得到一个特定的值。

function add(){
	var args = [...arguments];
	var fn = function(){
		var newArgs = [...args,...arguments];
		return add.apply(this,newArgs);
	}
	fn.toString = function(){
		return args.reduce((prev,curr)=>prev+curr);
	}
	return fn;
}

var a = add(1)(2)(3)(4); //  10
var b = add(1, 2, 3, 4); //  10
var c = add(1, 2)(3, 4); //  10
var d = add(1, 2, 3)(4); //  10
// 可以利用隐式转换的特性参与计算
console.log(a + 10); // 20
console.log(b + 20); // 30
console.log(c + 30); // 40
console.log(d + 40); // 50
// 也可以继续传入参数，得到的结果再次利用隐式转换参与计算
console.log(a(10) + 100); // 120
console.log(b(10) + 100); // 120
console.log(c(10) + 100); // 120
console.log(d(10) + 100); // 120
