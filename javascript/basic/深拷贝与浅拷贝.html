# 深拷贝与浅拷贝

# 概念

我理解的就是浅拷贝就是复制一层，深拷贝是通过递归全部复制。

# 浅拷贝

首先可以通过Object.assign来解决这个问题。具体解释参考这里。

let a = {
    age: 1
}
let b = Object.assign({}, a)
a.age = 2
console.log(b.age) // 1

当然我们也可以自己实现一个浅拷贝函数

function shallowCopy (obj) {//只拷贝一层
	// 只拷贝对象
	if(Object.prototype.toString.call(obj)!=='[object Object]') return;
	//判断是对象还是数组
	var newObj = Array.isArray(obj)?[]:{};
	for(var key in obj){
		if(obj.hasOwnProperty(key)){
			newObj[key] = obj[key];
		}
	}
	return newObj;
}

通常浅拷贝就能解决大部分问题了，但是当我们遇到如下情况就需要使用到深拷贝了

let a = {
    age: 1,
    jobs: {
        first: 'FE'
    }
}
let b = {...a}
a.jobs.first = 'native'
console.log(b.jobs.first) // native

浅拷贝只解决了第一层的问题，如果接下去的值中还有对象的话，那么就又回到刚开始的话题了，两者享有相同的引用。要解决这个问题，我们需要引入深拷贝。

# 深拷贝

这个问题通常可以通过 JSON.parse(JSON.stringify(object)) 来解决。

let a = {
    age: 1,
    jobs: {
        first: 'FE'
    }
}
let b = JSON.parse(JSON.stringify(a))
a.jobs.first = 'native'
console.log(b.jobs.first) // FE

但是该方法也是有局限性的：

  • 会忽略 undefined
  • 不能拷贝函数
  • 不能解决循环引用的对象
let obj = {
  a: 1,
  b: {
    c: 2,
    d: 3,
  },
}
obj.c = obj.b
obj.e = obj.a
obj.b.c = obj.c
obj.b.d = obj.b
obj.b.e = obj.b.c
let newObj = JSON.parse(JSON.stringify(obj))
console.log(newObj)

你会发现在上述情况中，该方法会忽略掉函数和 undefined 。

为此我们可以通过递归的方法进行遍历拷贝，即看遍历到的属性是对象还是属性，如果是对象的话就继续递归遍历。

function deepCopy (obj) {//递归拷贝多层
	// 只拷贝对象
	if(Object.prototype.toString.call(obj)!=='[object Object]') return;
		//判断是对象还是数组
		var newObj = Array.isArray(obj)?[]:{};
		for(var key in obj){
			if(obj.hasOwnProperty(key)){
				newObj[key] = Object.prototype.toString.call(obj[key]) =='[object Object]'?deepCopy (obj[key]):obj[key];
			}
		}
		return newObj;
	}

# 性能问题

尽管使用深拷贝会完全的克隆一个新对象，不会产生副作用，但是深拷贝因为使用递归，性能会不如浅拷贝，在开发中，还是要根据实际情况进行选择。
