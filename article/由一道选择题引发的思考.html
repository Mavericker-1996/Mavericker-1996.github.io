# Function构造器

Function 是一个构造器，能创建Function对象，即JavaScript中每个函数实际上都是Function 对象。

# 构造方法

new Function ([arg1[, arg2[, ...argN]],] functionBody)
1

# 参数

  1. arg1, arg2, ... argN

    被函数使用的参数的名称必须是合法命名的。参数名称是一个有效的JavaScript标识符的字符串，或者一个用逗号分隔的有效字符串的列表;例如“×”，“theValue”，或“A，B”。

  2. functionBody

    一个含有包括函数定义的JavaScript语句的字符串。

# 举例

var sum = new Function('a', 'b', 'return a + b');

console.log(sum(2, 6));
// expected output: 8
1
2
3
4

上述方法创建function等同于普通函数声明方法：

function sum(a,b){return a+b;}
1

# 属性

Function.arguments

以数组形式获取传入函数的所有参数。此属性已被arguments替代。

Function.caller

获取调用函数的具体对象。

Function.length

获取函数的接收参数个数。

Function.name

获取函数的名称。

Function.displayName

获取函数的display name。

Function.prototype

原型，最重要的属性。

# 方法

Function.prototype.apply()

在一个对象的上下文中应用另一个对象的方法；参数能够以数组形式传入。

Function.prototype.bind()

bind()方法会创建一个新函数,称为绑定函数.当调用这个绑定函数时,绑定函数会以创建它时传入 bind()方法的第一个参数作为 this,传入 bind()方法的第二个以及以后的参数加上绑定函数运行时本身的参数按照顺序作为原函数的参数来调用原函数.

Function.prototype.call()

在一个对象的上下文中应用另一个对象的方法；参数能够以列表形式传入。

Function.prototype.isGenerator()

若函数对象为generator，返回true，反之返回 false。

Function.prototype.toSource()

获取函数的实现源码的字符串。 覆盖了 Object.prototype.toSource 方法。

Function.prototype.toString()

获取函数的实现源码的字符串。覆盖了 Object.prototype.toString 方法。

# 题目分析

首先我们可以看出Women函数和Men函数中都定义了name属性。

然后通过Object.defineProperty方法修改了Women作为一个Function对象里面的name属性。

然后通过new方法声明了Women对象a和Men对象b。

然后通过原型继承的方法将a继承于b。

所以Women.name返回定义get函数中返回的内容，即yy3。

Men.name返回函数的名称Men。

由于new Women()传入的name值为空，所以a.name返回undefined。如果我们将Women函数中的this.name = name注释掉，由于继承的缘故，a.name就是yy1。

b.name返回yy1。
