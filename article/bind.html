# 模拟实现bind

参考文章：

JavaScript深入之bind的模拟实现

从一道面试题的进阶，到“我可能看了假源码”

由于bind的实现涉及到一些比较重要的概念，所以打算单独抽出来写一篇文章。

# bind是什么

一句话介绍 bind:

bind() 方法会创建一个新函数。当这个新函数被调用时，bind() 的第一个参数将作为它运行时的 this，之后的一序列参数将会在传递的实参前传入作为它的参数。(来自于 MDN )

由此我们可以首先得出 bind 函数的两个特点：

  1. 返回一个函数
  2. 可以传入参数（柯里化）

下面给出一个例子来说明这两点：

var foo = {
    value: 1
};

function bar(name, age) {
    console.log(this.value);
    console.log(name);
    console.log(age);

}

var bindFoo = bar.bind(foo, 'daisy');
bindFoo('18');
// 1
// daisy
// 18

# 第一版

结合这两个特点，我们便可以使用es6的语法写出第一版代码：

Function.prototype.bind2 = function(context, ...args1) {
    var func = this;
    return function(...args2) {
        return func.apply(context, [...args1, ...args2])
    } 
}

# 第一版问题

bind还有一个特点，就是

一个绑定函数也能使用new操作符创建对象：这种行为就像把原函数当成构造器。提供的 this 值被忽略，同时调用时的参数被提供给模拟函数。

也就是说当 bind 返回的函数作为构造函数的时候，bind 时指定的 this 值会失效，但传入的参数依然生效。举个例子：

var value = 2;

var foo = {
    value: 1
};

function bar(name, age) {
    this.habit = 'shopping';
    console.log(this.value);
    console.log(name);
    console.log(age);
}

bar.prototype.friend = 'kevin';

var bindFoo = bar.bind(foo, 'daisy');

var obj = new bindFoo('18');
// undefined
// daisy
// 18
console.log(obj.habit);
console.log(obj.friend);
// shopping
// kevin

# 第二版

为了解决作为构造函数的问题，我们可以通过instanceof来判断this的指向，如果是作为构造函数的话，this指向实例，所以在实例的__proto__上找到前面返回函数的prototype，否则就还是像之前一样绑定传过来的context。

对于访问原函数prototype属性的问题，我们可以采用寄生组合继承中的思想，通过Object.create方法来实现，也可以自己模拟实现Object.create方法。

最后我们再加上一个对调用bind的对象的类型判断，如果不是函数的话，我们要报错。

修正后的代码如下：

Function.prototype.bind3 = function (context, ...args1) {
	if (typeof this !== "function") {
  		throw new Error("Function.prototype.bind - what is trying to be bound is not callable");
	}
	var func = this;
	var fBound = function (...args2) {
		return func.apply(this instanceof fBound ? this : context, [...args1, ...args2])
	}
	fBound.prototype = Object.create(func.prototype);
	return fBound;
}
