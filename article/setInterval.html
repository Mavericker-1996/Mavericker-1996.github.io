# 解决setInterval计时不准的问题

参考文章：

JS实现活动精确倒计时

解决setInterval计时器不准的问题

# 问题原因

在js中如果打算使用setInterval进行倒数，计时等功能,往往是不准确的，因为setInterval的回调函数并不是到时后立即执行，由于js是单线程的缘故，而是等系统计算资源空闲下来后才会执行。而下一次触发时间则是在setInterval回调函数执行完毕之后才开始计时，所以如果setInterval内执行的计算过于耗时，或者有其他耗时任务在执行，setInterval的计时会越来越不准,延迟很厉害。

# 举例说明

下面的代码可以说明这个问题

var startTime = new Date().getTime();
var count = 0;
//耗时任务
setInterval(function(){
    var i = 0;
    while(i++ < 100000000);
}, 0);
//测试任务
setInterval(function(){
    count++;
    console.log(new Date().getTime() - (startTime + count * 1000));
}, 1000);

其中我们测试的setInterval周期是1s，然后我们定义了startTime作为开始的时间，count作为用于计数的变量，并且只有在测试任务执行的时候才会加1。所以startTime+count*1000就是应该触发的准确时间，再用当前时间减去的话就是 误差时间了。

# 校准时差

为了在js里可以使用相对准确的计时功能，我们可以用setTimeout代替setInterval，然后算出下次触发的时间，从而修正了当前触发的延迟。

var startTime = new Date().getTime();
var count = 0;
setInterval(function(){
    var i = 0;
    while(i++ < 100000000);
}, 0);
function fixed() {
    count++;
    var offset = new Date().getTime() - (startTime + count * 1000);
    var nextTime = 1000 - offset;
    if (nextTime < 0) nextTime = 0;
    setTimeout(fixed, nextTime);
     
    console.log(new Date().getTime() - (startTime + count * 1000));
}
setTimeout(fixed, 1000);
