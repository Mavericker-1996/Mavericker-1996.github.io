数组 双指针

参考文章：leetcode之3sum的js实现，015. 3Sum

# 解题思路

3个数的和为0。在我们确定a的值的情况下，我们只需找到b+c = -a的两个数字。首先，我们不可能遍历所有2个数字的组合，因为随着问题规模n的增大，算法的时间复杂度会急剧增高。所以我们想到，在有序数列的情况下，采用双指针的线性时间复杂度来解决这个问题。

所谓的双指针方法，即指在有序数列中，我们在头尾分别设定两个指针，然后根据相关条件来不断向中间靠拢，这样一来我们只需要遍历一次就可以结束。从而避免了双重循环带来的复杂度问题。

所以我们要指定数组nums升序

然后我们假设a=nums[k]，当a > 0的时候，nums[k+n]均大于0，此时计算应该停止，因为绝对不存在a+b+c = 0的情况了。

如果nums[k] == nums[k-1] (k > 0)，那么这时候要跳过本次循环，因为要去重。

# 代码实现

/**
 * @param {number[]} nums
 * @return {number[][]}
 */
var threeSum = function(nums) {
	nums.sort((a,b) => a-b); //先给输入数组排序
    var res = []; //储存结果数组
    for (var i=0; i<nums.length-1; i++) {
    	if (nums[i] > 0) break; //后面的数都大于零，直接跳出整个循环
    	if (i > 0 && nums[i] == nums[i-1]) continue; //当前数和上一个数相等的话，跳出本次循环防止重复

    	var m = i+1, n = nums.length - 1, target = -nums[i]; //在有序数列的情况下，采用双指针的线性时间复杂度来解决这个问题
    	while (m < n) {
    		if (nums[m] + nums[n] == target) {
    			res.push ([nums[i], nums[m], nums[n]]);
    			while (m < n && nums[m] == nums[m+1]) m++;
    			while (m < n && nums[n] == nums[n-1]) n--;

    			m++;
    			n--;
    		} else if (nums[m] + nums[n] > target) n--;
    		  else m++;
    	}
    }
    return res;
};

# 最接近的三数之和

# 题目描述

给定一个包括 n 个整数的数组 nums 和 一个目标值 target。找出 nums 中的三个整数，使得它们的和与 target 最接近。返回这三个数的和。假定每组输入只存在唯一答案。

例如，给定数组 nums = [-1，2，1，-4], 和 target = 1.

与 target 最接近的三个数的和为 2. (-1 + 2 + 1 = 2).
1
2
3

# 相关话题

数组 双指针

参考文章：16. 3Sum Closest [M]

# 解题思路

思路与三数之和大致相同，需要用两个变量分别存储返回的结果min，和当前的相加结果tmpSum，通过比较两者与target的差的绝对值来进行赋值操作，如果min直接等于target，则直接返回，否则继续遍历。

# 代码实现

/**
 * @param {number[]} nums
 * @param {number} target
 * @return {number}
 */
var threeSumClosest = function(nums, target) {
	nums.sort((a, b) => a - b); //先给输入数组排序
	var min = nums[0] + nums[1] + nums[2]; //返回的结果
	for (var i = 0; i < nums.length - 1; i++) {
		var m = i + 1,
			n = nums.length - 1,
			tmpSum,
			tmpTarget = target - nums[i]; //在有序数列的情况下，采用双指针的线性时间复杂度来解决这个问题
		while (m < n) {
			tmpSum = nums[i] + nums[m] + nums[n]; 
			if (Math.abs(target - tmpSum) < Math.abs(target - min)) {
				min = tmpSum;
			} 
			if (min == target) return min;
			else if (nums[m] + nums[n] > tmpTarget) n--;
			else m++;
		}
	}
	return min;
};

# 合并区间

(美团一面，猿辅导一面都有问到)

# 题目描述

给出若干闭合区间，合并所有重叠的部分。 示例如下：

[[8,10],[15,18],[2,6],[3,5],[1,3]] => [[1,6], [8,10], [15,18]]
1

# 相关话题

排序 数组

# 解题思路

首先把给出的数组中每一项按照元素的第一个数大小进行排序，如果第一个数相等的话，就比较第二个数大小。

然后通过reduce方法依次比较前后两个数组：

如果满足pre[1]>=curr[0]的条件时即可合并，这里需要注意的是合并后的第二个数要取Math.max(pre[1],curr[1])，即两者中比较大的一个，因为有可能出现[1,5],[2,4]=>[1,5]的情况。这时就接着往下比较，不用往结果数组中push，直接返回合并后的数组作为下一次循环的prev。

如果不满足合并条件，我们就将上一项push到结果数组中，然后返回当前项，作为下一次循环的prev，继续比较。

比较到最后我们会发现我们没有对最后一项进行处理，这时我们可以将reduce到最后的结果存起来，然后push到结果数组中。

最后我们返回结果数组即可。

# 代码实现

const merge = arr => {
	arr = arr.sort( (a,b) => a[0] - b[0]); 
	console.log(arr);
	var res = [];
	var last = arr.reduce((prev,curr) => {
		if(prev[1] >= curr[0]) {
			return [prev[0], Math.max(prev[1], curr[1])];
		} else {
			res.push(prev);
			return curr;
		}
	})
	res.push(last);
	return res;
}

var arr = [[8,10],[15,18],[2,6],[3,5],[1,3]];;
console.log(merge(arr))
