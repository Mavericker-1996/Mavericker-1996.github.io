# Rekit Demo

最近跟着王沛老师的React教程学了快四分之一了，掌握了React，Redux的基础操作。准备借着课程中的两个简单demo把之前学的知识整理一下。

# 关于Rekit

官方文档介绍如下：

Rekit is a toolkit for building scalable web applications with React, Redux and React-router. It's an all-in-one solution for creating modern React apps.

It helps you focus on business logic rather than dealing with massive libraries, patterns, configurations etc.

简单来说就是一个可以帮助您专注于业务逻辑，而不是处理大量的库，模式，配置等的开发环境，其中有两个突出的特点：

  1. feature oriented architecture （面向功能的体系结构）
  2. one action per file pattern （一个action一个文件）

具体解释可以点击链接进行查看。这样的好处是可确保应用程序逻辑很好地分组和解耦。

对于这些action文件中的reducer是怎么整合在一起的，我们可以在Misc/redux/reducer.js找到：

在这里引用作者对这段代码的解释：

From the code we can see: the wrapper reducer itself is also a place to write reducers. It and reducers from actions operates on the same branch of the store. This is the key difference between a standard reducer with the one defined in an action file. You can think a wrapper reducer as a standard Redux reducer while others are just pure functions.

关键在于最后一行代码：

return reducers.reduce((s, r) => r(s, action), newState);
1

这行代码的意思就是把分支的reducer当成纯函数来执行，并以newState作为初始值。并将上一个返回的state作为下一个reducer函数的输入参数，包裹在一个reducer中，相当于一个总的reducer函数。

# Demo2-fetchRedditList

第二个demo目的就是练习一下异步action的操作。目的是获得reddit上与reactjs相关的文章并显示出来。我们首先新建一个组件，然后再新建一个异步的action，打开action文件已经为我们写好大部分代码了。Rekit是使用redux-thunk作为异步action的中间件的。关于redux-thunk，可以参考这篇和这篇文章。引用阮一峰老师的总结就是：

因此，异步操作的第一种解决方案就是，写出一个返回函数的 Action Creator，然后使用redux-thunk中间件改造store.dispatch。

首先它已经为我们定义好了异步请求的四种状态：

  • DEMO_FETCH_REDDIT_LIST_BEGIN,
  • DEMO_FETCH_REDDIT_LIST_SUCCESS,
  • DEMO_FETCH_REDDIT_LIST_FAILURE,
  • DEMO_FETCH_REDDIT_LIST_DISMISS_ERROR

接着我们就可以进行action异步操作了，其中fetchRedditList相当于一个action creator，只不过它返回的是一个函数而不是一个action对象，这里redux-thunk会自动帮我们处理。代码如下，其中返回的函数接受两个方法：dispatch和getState，我们可以根据promise返回的状态来dispatch不同类型的action。之所以返回promise下面的注释有提到，目的是方便我们进行接下来的操作，比如提交表单之后进行重定向或者返回失败信息。

export function fetchRedditList(args = {}) {
  return dispatch => {
    // optionally you can have getState as the second argument
    dispatch({
      type: DEMO_FETCH_REDDIT_LIST_BEGIN,
    });

    // Return a promise so that you could control UI flow without states in the store.
    // For example: after submit a form, you need to redirect the page to another when succeeds or show some errors message if fails.
    // It's hard to use state to manage it, but returning a promise allows you to easily achieve it.
    // e.g.: handleSubmit() { this.props.actions.submitForm(data).then(()=> {}).catch(() => {}); }
    const promise = new Promise((resolve, reject) => {
      // doRequest is a placeholder Promise. You should replace it with your own logic.
      // See the real-word example at:  https://github.com/supnate/rekit/blob/master/src/features/home/redux/fetchRedditReactjsList.js
      // args.error here is only for test coverage purpose.
      // const doRequest = args.error ? Promise.reject(new Error()) : Promise.resolve();
      const doRequest = axios.get('https://www1.reddit.com/r/reactjs.json');
      doRequest.then(
        res => {
          // console.log(res);
          dispatch({
            type: DEMO_FETCH_REDDIT_LIST_SUCCESS,
            data: res.data.data.children.map(item => item.data),
          });
          resolve(res);
        },
        // Use rejectHandler as the second argument so that render errors won't be caught.
        err => {
          console.log(err);
          dispatch({
            type: DEMO_FETCH_REDDIT_LIST_FAILURE,
            data: { error: err },
          });
          reject(err);
        },
      );
    });

    return promise;
  };
}

接下来我们来看一下reducer函数，我们只需要不同状态返回不同的new state就好：

export function reducer(state, action) {
  switch (action.type) {
    case DEMO_FETCH_REDDIT_LIST_BEGIN:
      // Just after a request is sent
      return {
        ...state,
        fetchRedditListPending: true,
        fetchRedditListError: null,
      };

    case DEMO_FETCH_REDDIT_LIST_SUCCESS:
      // The request is success
      return {
        ...state,
        fetchRedditListPending: false,
        fetchRedditListError: null,
        redditList: action.data,
      };

    case DEMO_FETCH_REDDIT_LIST_FAILURE:
      // The request is failed
      return {
        ...state,
        fetchRedditListPending: false,
        fetchRedditListError: action.data.error,
      };

    case DEMO_FETCH_REDDIT_LIST_DISMISS_ERROR:
      // Dismiss the request failure error
      return {
        ...state,
        fetchRedditListError: null,
      };

    default:
      return state;
  }
}
